# Codex 岗位语义分析任务

此文件是每日 Codex 自动任务的执行规范。模型只分析公开招聘数据，不修改个人档案。

## 输入与增量规则

1. 运行 `scripts/prepare_codex_analysis.py` 生成 `data/ai-pending.json`。
2. 只分析 pending 中的记录；`content_hash` 未变化的结果必须复用。
3. 每完成一批就合并写回 `data/ai-analysis.json`，不得清空既有结果。
4. 原文没有说明的条件必须标记为“不明确”，禁止猜测。
5. Python 采集器只保存网页正文、附件、工作表原始列名与原始单元格值，不做字段映射、条件抽取或匹配判断。模型必须直接理解 `body_text`、`raw_fields` 和附件岗位原文，并自行完成全部字段归一化。
6. 截止时间由 Codex 从原文语义归一化到 `normalized.deadline`，页面以该值为准。原文明示具体时刻时必须保留时分和中国时区，例如“7月7日18:00”写为 `2026-07-07T18:00:00+08:00`；原文未说明才留空。

## 匹配标准

个人条件来自 `data/profile.json`。综合理解标题、公告正文、岗位要求、附件字段和职责：

- 明确冲突（毕业年份、学历、专业、性别等）为 `no`。
- 关键条件明确满足且没有冲突为 `match`。
- 关键条件缺失、表述模糊或需要查专业代码为 `possible`。
- “优先”不是硬性限制；“限”“仅限”“必须”才按硬条件处理。
- 岗位明确列出专业且不包含设计类、专业不限或合理的艺术设计相邻方向时，直接判为 `no`，不得标记为“专业需确认”。
- 软件开发、后端、算法、测试开发、运维、数据工程等岗位，若原文职责或专业明确要求计算机、软件工程等技术背景且未接受设计类专业，直接判为 `no`；不得仅因某个字段缺失而降级为 `possible`。
- 对“互联网大厂”岗位，不得因为没有专业代码就默认标记为 `possible`。必须重点理解 Job Description、工作职责、任职要求、技能栈、作品集要求、招聘类型和工作年限，再判断岗位与个人条件的真实匹配程度。
- 用户体验、UI、UX、交互、视觉、品牌、创意、产品设计、工业设计、动效、设计研究、设计系统等以设计产出为核心的职责，可判定为设计相关；仍须继续核对届别、学历、经验和其他硬条件。
- 内容运营、产品运营、新媒体、视频制作、用户研究、产品经理等相邻岗位，必须根据日常核心职责和硬性技能判断：设计能力是主要工作内容时可保留，设计仅为加分项或偶尔协作时不能据此判为符合。
- 编程开发、系统架构、算法建模、服务端、客户端、测试开发、数据平台、网络安全等以工程交付为核心的职责，与设计背景明确无关时直接判为 `no`，即使岗位名称宽泛或未列专业也不得保留。
- 大厂社会招聘岗位若明确要求全职工作年限、资深职级或专家经验，与2027届应届生冲突时直接判为 `no`；校招和实习岗位则结合可入职时间、毕业年份和实习时长判断。
- 大厂岗位的判断必须至少引用一条来自职责或任职要求的简短原文证据。仅凭岗位名称、公司名称或 Python 提取字段不得下结论。
- 执业医师、住院医师规范化培训、护士执业资格、法律职业资格等与设计背景无关的强制资格，直接判为 `no`。
- 明确要求一年及以上全职工作经验，或岗位为资深、高级、专家、负责人、架构师、总监时，按应届生硬冲突判为 `no`；学生干部或实习经历不等同于全职工作经验。
- 设计相关语义包括但不限于设计学、艺术设计、视觉传达、产品设计、交互设计、用户体验、数字媒体艺术、环境设计、工业设计等；仍需结合原文判断。
- 专业目录须同时兼容两个版本：2018版 `1305 设计学` 与2022版 `1357 设计` 均按设计专业明确匹配。
- 招聘条件写明2022版 `1301 艺术学` 且同时明确设计方向时按匹配处理；若只写 `1301 艺术学`、未限定方向，则标记为 `possible`，不能仅凭该代码判定明确符合或不符合。
- 招聘条件只写上位门类 `13`、`艺术学门类（13）` 或未限定方向的 `1301` 时，可能涵盖设计专业，必须保留为 `possible` 供人工查看，不得筛除。
- 模型必须自行识别附件中的列名语义，例如“专业要求”“对外发布公告专业要求”“所学专业”等；不得依赖或要求 Python 将其映射成标准字段。
- `possible` 的 `needs_confirmation` 必须写出具体待确认事项及原因，例如“专业仅写1301艺术学，需确认是否接收设计方向”或“未注明是否接收2027届”；禁止只写“查看公告”“信息不足”等空泛表述。
- 必须给出简短原文证据，不得编造学校层次、专业归属或招聘条件。
- 截止时间的判断必须附带含日期或报名区间的简短原文证据；“发布日”“资格审查日”等不得误当报名截止日。

## 完成与发布

更新顶层 `generated_at`、`profile_version` 和 `prompt_version`。重新运行准备脚本确认剩余数量，构建网站，提交并推送分析结果。若没有新增或变化岗位，不改文件、不创建空提交。
